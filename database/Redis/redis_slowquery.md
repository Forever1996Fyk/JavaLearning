## Redis-慢查询

我们知道MySQL提供了慢查询日志, 可以定位系统存在的慢操作, Redis中也提供了类似的功能。

所以慢查询日志就是系统记录执行时间超过阈值的命令, 包括发生时间, 耗时, 命令的详细信息。

慢查询的作用: **通过慢查询分析, 找到有问题的命令进行优化。**

Redis执行命令一般分为4个步骤: 

1. 发送命令

2. 命令排队

3. 执行命令

4. 返回结果。

而慢查询只统计执行命令的时间, 所以没有慢查询, 客户端也有可能超时。

具体慢查询的详细介绍, 可以看这篇文章---[如何排查 Redis 中的慢查询](http://cmsblogs.com/?p=18352)
